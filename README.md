# tech-support-topic-classification
Классификация обращений в техническую поддержку по тематическим категориям с использованием нейронных сетей на PyTorch.

# Классификация обращений в техническую поддержку

Проект посвящён автоматической классификации текстовых обращений пользователей в техническую поддержку по тематическим категориям с использованием нейронных сетей.

В работе рассматривается задача многоклассовой классификации текстов. На основе текста обращения модель определяет, к какой группе относится запрос пользователя: проблемы с оборудованием, доступами, хранилищем, административными правами, закупками и другими внутренними процессами.

## Цель проекта

Цель работы — построить и сравнить несколько нейросетевых моделей для классификации обращений в техподдержку и определить, какая архитектура показывает наилучшее качество на данной задаче.

## Данные

В проекте используется датасет `tech_support_topics.csv`, содержащий тексты обращений и соответствующие им тематические категории.

Целевая переменная:

- `Topic_group` — категория обращения.

В датасете представлены следующие классы:

- Hardware
- HR Support
- Access
- Miscellaneous
- Storage
- Purchase
- Internal Project
- Administrative rights

Всего в наборе данных около 48 тысяч обращений.

## Этапы работы

В ходе проекта были выполнены следующие шаги:

1. Загрузка и первичный анализ данных.
2. Проверка распределения обращений по тематическим категориям.
3. Кодирование целевых меток с помощью one-hot encoding.
4. Токенизация текстов обращений.
5. Приведение текстовых последовательностей к фиксированной длине.
6. Разделение данных на обучающую и тестовую выборки.
7. Реализация нескольких нейросетевых моделей на PyTorch.
8. Обучение моделей и отслеживание метрик качества.
9. Сравнение результатов моделей.
10. Выбор лучшей архитектуры для задачи классификации.

## Используемые модели

В работе были реализованы и обучены три модели:

### 1. LSTM

Модель на основе рекуррентной нейронной сети LSTM.

Архитектура:

- Embedding layer
- LSTM
- Global Max Pooling
- Fully Connected слой
- Выходной слой классификации

Эта модель показала лучший результат среди рассмотренных архитектур.

### 2. Bidirectional LSTM

Двунаправленная LSTM-модель, которая обрабатывает текстовую последовательность в прямом и обратном направлениях.

Такой подход позволяет учитывать контекст как слева направо, так и справа налево. Однако в данной задаче прирост качества по сравнению с обычной LSTM оказался незначительным.

### 3. CNN / Conv1D-модель

Сверточная модель для обработки текста.

Архитектура включает несколько одномерных сверточных слоев, которые извлекают локальные признаки из текстовых последовательностей.

Модель показала более низкое качество по сравнению с LSTM-подходами.

## Результаты

По итогам экспериментов были получены следующие результаты:

| Модель | Accuracy |
|---|---:|
| LSTM | ~0.839 |
| Bidirectional LSTM | ~0.835 |
| CNN / Conv1D | ~0.793 |

Лучший результат показала LSTM-модель с точностью около 84%.

## Вывод

В рамках проекта была решена задача классификации обращений пользователей в техническую поддержку. Были реализованы и сравнены несколько нейросетевых архитектур для обработки текстовых данных.

Наилучшее качество показала LSTM-модель. Bidirectional LSTM дала близкий результат, но не обеспечила заметного улучшения. Сверточная модель оказалась менее эффективной для данной задачи.

Таким образом, для классификации обращений в техподдержку на данном датасете наиболее подходящей оказалась рекуррентная архитектура LSTM.
